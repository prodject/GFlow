GControl coverage log
Updated: 2026-07-21

Source inventory
- .example: old Monji APK decompile and bundled resources.
- .src/Functions.txt: functional target list extracted from old app.
- .src/Ext: system APKs and decompiled sources used as integration references.

Key system sources found in .src/Ext
- OneOS-SystemUIPlugin_src: com.ecarx.xui.adaptapi.* including car, hvac, vehicle, dvr, evs, hudinteraction, tpms, wifiap, audiofx.
- ECarXCarService_src: ecarx car service and permission ecarx.car.permission.CAR_HVAC.
- CarService_src: Android Automotive car service permissions and vehicle categories.
- DIMInteraction_src: DIM/QNX interaction, navigation/phone/camera monitor classes.
- AudioExtService_src: autolink InteractionProto messages for dock/DIM/audio/media/navigation.
- OneOS-Dock_src and OneOS-ControlBoard_src: OneOS UI/cloud/navigation/dock integration classes.
- MultiWindowService_src and MultiWindowAnimator_src: privileged split/multi-window behavior.
- TCAMService_src: car vendor extension service entry points.

Coverage status legend
- implemented: functional code exists in GControl.
- partial: user-facing UI exists and some system/API calls are attempted.
- adapter: reflection-based system contract added; requires target firmware classes/permissions.
- scaffold: entry point/service exists but real behavior is not implemented.
- missing: not implemented yet.
- privileged: requires system signature, root, or adb grants.
